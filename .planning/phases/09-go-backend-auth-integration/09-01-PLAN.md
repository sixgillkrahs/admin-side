# Plan 09-01: Go Backend Auth Integration - Login API & State Setup

## Objective
Connect credentials verification and token-based session retrieval via POST `/api/v1/auth/login` to the admin panel using Zustand store state (API-01).

## User Story
**As an** administrator, **I want to** log in with my email and password, **so that** I can retrieve my JWT session token and user info, persisting it securely in Zustand state.

## Verification Checklist

### Automated Tests
- Running `npm run build` in frontend/admin-side succeeds.
- Run Go backend server and verify that OPTIONS preflight requests return status `204`.

### Manual Verification
- Attempt to login with invalid credentials and verify that the red error banner displaying the correct translation of "The email or password you entered is incorrect" appears.
- Attempt to login with valid credentials (e.g. `admin@example.com` / `securepassword` or as configured in Go seeds) and verify that the user is redirected to the dashboard.
- Verify that the `antigravity-auth-storage` key is set in Session Storage with correct user and token details.

## Goals and must_haves

### Goals
- Secure credentials-based authentication integration with Zustand state.

### must_haves
- Submitting valid credentials requests POST `/api/v1/auth/login` and retrieves a JWT access token.
- JWT access token is stored in sessionStorage via Zustand persist middleware.
- Login screen shows localized error warnings for incorrect credentials and server connection failures.
