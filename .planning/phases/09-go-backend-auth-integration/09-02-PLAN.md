# Plan 09-02: Go Backend Auth Integration - Interceptor & IP/Session Warnings

## Objective
Implement client IP verification response handling and centralize `401 Unauthorized` token failure catching to clear session data and redirect to the login screen with localized warnings (API-02).

## User Story
**As an** administrator, **I want** my session to be validated for IP lock integrity, and **I want to** be redirected to the login screen with clear, localized warning messages if my session expires or if my client IP address changes.

  1. The user is logged out (the token is cleared from Session Storage).
  2. The user is redirected to the login screen.

## Goals and must_haves

### Goals
- Automated session tracking validation and secure fallback redirection with translation support.

### must_haves
- Outgoing requests automatically inject JWT token headers.
- Status `401 Unauthorized` token failures clear Zustand auth store immediately.
- User is redirected back to LoginScreen on token failures.
- Session warning banner shows specific localized messages for IP lock mismatches and token expirations.
